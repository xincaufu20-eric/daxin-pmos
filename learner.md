# Learner.md - Claude 与我的协同学习日志

> 这是 Claude 自己维护的学习日志。
> 当我反复修改 Claude 的某类输出时，Claude 会在这里记录差异，
> 累积到一定次数后建议更新对应 SKILL.md。
>
> 灵感来源：Mahesh Yadav 的 PM Builder 方法论。

---

## 工作原理

```
我修改 Claude 的输出
    ↓
Claude 把"我的修改"和"原输出"的差异记到这里
    ↓
同一类修改累积 5 次
    ↓
Claude 主动建议："要不要把这个偏好加到 CLAUDE.md 或 XX skill？"
    ↓
我确认 → Claude 自动更新对应文件
    ↓
下次默认就这么输出
```

### 2026-06
**实地调研 42 分钟空转事件复盘**

一次完整产品考古中,42 分钟被子智能体空转吃掉。三个根因:
1. 子智能体的工具权限和主智能体不一样,但派出前没验证 —— 最大头(占 80% 时间损失)
2. WebFetch 拦截评论站后没立即降级 WebSearch,在同域名上反复试
3. 多次用"我现在并行做 X"包装话术,但没真正发起工具调用,话和动作分离

已沉淀为 `CLAUDE.md` →「🛠️ 工具调度纪律(强制)」三条规则。下次产品走查前先复读这一条。

**反直觉判断**:表面看是 Playwright MCP 配置慢拖了进度(9 分钟),实际是子智能体调度策略错了(42 分钟)。Playwright 是一次性投入,下次免费;但"派子智能体不验权限"是个会反复犯的系统性错误——不改的话,每次实地调研都会再丢 30-60 分钟。
