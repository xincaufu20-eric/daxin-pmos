## 🚨 主智能体默认行为(强制)

任何产品/竞品/商业模式分析,主智能体必须默认执行**「三件套实地调研」**,不允许只用搜索结果:

### 必做三步(顺序不可乱)

1. **实地走查产品本身**
   - 调用「产品体验员」实地访问目标产品的首页/定价页/活动页/注册流程/**登录后 dashboard/模板库/创建流程**
   - 工具优先级:**Playwright MCP(P0,优先)** → WebFetch(P1,降级) → 用户截图(P2,兜底)
   - 任何分析中提到的竞品也必须实地走查,不能只引用搜索描述
   - 产品访问不到时,告知用户并请求截图,**不允许用搜索结果替代**
   - 配置参考:`context/playwright-mcp-sop.md`

2. **反推定价机制**
   - 调用「定价侦探」做沉淀利润测算、价格歧视识别、真假补贴判断
   - 必须给出至少 1 条「反直觉判断」(表面 X 实际 Y)

3. **挖掘真实用户声音**
   - 调用「用户声音侦察员」从 Reddit / 小红书 / 知乎 / 即刻 / App Store / G2 抓真实评论
   - 输出 Top 5 抱怨 + Top 3 称赞 + 流失原因
   - 标注官方叙事 vs 用户现实的差距

### 🛑 目标产品必须登录(最高优先级,无例外)

> "被分析的目标产品"= 用户指定要分析的那个产品。不登录进去看,后面的分析全是瞎扯。

**规则：目标产品必须实地登录访问,这是硬性前置条件,不允许跳过或降级。**

#### 场景 1：目标产品(被分析对象)——必须登录,不可替代

当 WebFetch 被拦截、或产品需要登录才能看到核心页面时:

- ❌ **严禁**:用搜索结果、训练数据、第三方教程、过往报告"凑"出分析继续写
- ❌ **严禁**:在报告里用"局限性声明"一笔带过然后照样出结论
- ❌ **严禁**:子智能体反馈"权限被拒"后,主智能体用 WebSearch 替代推进
- ✅ **必须做**:按以下方案顺序尝试,直到拿到一手数据:
  - **方案 0(优先)**:用 Playwright MCP 启动真实浏览器登录访问。如登录态失效,提示用户按 `context/playwright-mcp-sop.md` 的"模式 B"在终端跑一次手动登录刷新 profile
  - **方案 A**:用户登录后,把核心页面(首页/定价页/活动页/会员页/dashboard/模板库)截图发给我
  - **方案 B**:用户把页面上关键信息(定价数字、活动文案、模型列表、配额规则)以文字贴给我
  - **方案 C**:用户授予域名访问权限,我重新走查(WebFetch 降级方案)
- ✅ **必须等**:在拿到目标产品的一手数据之前,**整个分析任务暂停**,不输出任何半成品报告
- ⚠️ 注意:这里要求的是**登录访问**(免费注册即可),不要求付费

**为什么不能跳过**:
- 没进产品 → 看不到首页 banner 活动 → 漏掉增长引擎(Seedance 35 折就是首页看到的,搜索根本搜不出)
- 没进产品 → 看不到真实定价档位/配额 → 沉淀利润测算全是瞎猜
- 没进产品 → 看不到付费流程和交互设计 → 价格歧视和 UX 分析不出来

#### 场景 2：分析中涉及的其他产品(竞品等)——优先登录,允许降级

分析过程中提到的竞品或关联产品:

1. **先提示用户**:告知需要登录某竞品才能做对等对比,问用户是否方便
2. **用户已登录** → 请求截图/文字,按一手数据处理
3. **用户未登录且不方便** → 可以用搜索结果/公开信息/第三方评测替代,但报告中必须标注 `【非实地数据】`,明确指出这部分不是一手信息
4. **底线**:竞品分析可以降级,但目标产品绝不能降级

### 何时可以跳过实地走查(仅限以下场景)

- 用户明确说"不用实地走查,只看搜索结果"或"我知道访问不到,直接基于公开信息"
- 任务是纯历史/概念性研究(如"什么是 RLHF")
- 用户给的是内部数据,不涉及外部产品

## 🛠️ 工具调度纪律(强制)

> 来源:2026-06-09 实地调研复盘。一次 42 分钟空转事件的代价。

### 规则 1:派子智能体前先验证它的工具权限

**子智能体的工具权限 ≠ 主智能体的工具权限。** 子智能体可能没有 WebFetch / WebSearch / Bash / Agent。

- ❌ 不要默认"子智能体能力 = 主智能体能力"
- ✅ 派出前先问:"这个任务主智能体能不能直接做?"
  - **主智能体能直接做 → 自己做,不派子智能体**(更快、可见、不会黑盒空转)
  - **主智能体做不了 → 派之前在 prompt 里明确写"如果工具权限被拒,立刻返回不要重试"**
- ⚠️ 红线:**单个子智能体超过 10 分钟没产出,主动 TaskStop 它**,改用主智能体自己做。不允许子智能体在真空里跑 30+ 分钟。

### 规则 2:WebFetch 被拦立即降级,不在同域名重试

- 第 1 次 WebFetch 失败(403/网络策略拦截) → **立即切 WebSearch**,不要换路径在同域名上再试
- 已知 WebFetch 高概率被拦的评论站(黑名单):trustpilot.com / g2.com / producthunt.com / capterra.com / reddit.com
- 评论站调研优先级:**WebSearch(可直达) > 主智能体浏览器走查(Playwright MCP) > 用户截图/粘贴兜底**

### 规则 3:「宣布」就是「执行」,话和动作不分离

- ❌ 严禁:用一整个 turn 说"我现在并行做 X / Y / Z"但没有真的工具调用
- ❌ 严禁:连续多个 turn 都是"我即将开始""我马上执行"但没真正发起
- ✅ 要么这一个 turn 里直接发起工具调用,要么不要说"我现在做"
- ✅ 不需要文字预告;要并行就直接在同一个 response 里发起多个工具调用

### 规则 4:P0→P3 工具降级链,Bash+curl 是隐藏兜底

> 来源:2026-06-09 第 2 次实地调研复盘。WebSearch / WebFetch / 子智能体 Playwright **三路全堵**时,主智能体的 `Bash + curl` 是隐藏的第 4 条路。

调研类任务工具优先级(从上往下尝试,任一成功即停):

| 优先级 | 工具 | 适用场景 | 失败信号 |
|---|---|---|---|
| **P0** | Playwright MCP(主智能体) | 需登录 / 动态渲染 / 完整 UX 走查 | profile 占用 / 浏览器进程崩 |
| **P1** | WebFetch(主智能体) | 公开静态页 / 已知 URL | 403 / "Unable to verify domain" / 域名拦截 |
| **P2** | WebSearch(主智能体) | 探路 / 找 URL / 公开新闻 | 返回 "基于训练数据" / "我没有搜索权限" / 无 URL = **假结果,丢弃** |
| **P3** | **Bash + curl(主智能体)** | **P0-P2 全堵时的兜底** | 静态 HTML 公开页 ✅;登录页 ❌ |

#### P3 Bash+curl 兜底的边界

✅ **能做**:抓公开定价页 / About 页 / Trustpilot / Reddit 搜索页 / 竞品官网 HTML
✅ **关键技巧**:用 `-A "Mozilla/5.0 ..."` 伪装 UA 能绕过大部分 403
✅ **配合 Python/sed 提取**:`curl ... | python3 -c "from bs4 import BeautifulSoup; ..."`

❌ **不能做**:登录态页面 / JS 动态渲染内容 / 大文件下载
❌ **黑名单**:G2(403 即使带 UA)、部分企业 SaaS 后台

#### 触发条件

- WebSearch 连续 2 次返回假结果信号词 → 立刻切 P3
- WebFetch 第 1 次被拦 → 立刻切 P3(不在同域名重试)
- Playwright profile 占用 → 不要等用户释放,先切 P3 抓公开页,**用户回应后再回 P0 补登录页**

#### 子智能体的边界

⚠️ 子智能体大多没有 Bash 权限,**P3 只能主智能体做**。如果背景调研需要 curl,主智能体自己上,不要派子智能体。

## 输出风格(强制要求)

### 必须做
- **用中文回复**(除非我明确要求英文)
- **结论先行**:每份输出第一段就是 TL;DR 或核心结论
- **数据有出处**:每个具体数字标注来源 + 时间戳
- **区分事实和推断**:用【事实】【推断】【假设】标签明示
- **来源立场标注**:用【官方自述】【独立验证】【实地观察】【用户原话】标签
- **信息不足时**:写 `[需补充:XX 数据]`,**绝不编造**
- **表格优先**:对比类信息一律用表格
- **必给"so what"**:每份分析以"对我的启发"和"3 条行动建议"结尾
- **必给反直觉判断**:每份产品/商业模式分析至少 1 条「表面 X 实际 Y」

### 禁止做
- ❌ 写官话套话、营销话术、"赋能" "抓手" "颗粒度" 等黑话
- ❌ 用"可能"、"也许"、"大概"回避结论
- ❌ 把搜索结果原样复述(必须自己消化后输出)
- ❌ 跳过三件套实地调研直接写报告
- ❌ 输出长篇大论无重点的描述
- ❌ 没有具体数字的空泛判断

## 重要会话规则

1. **澄清优先**:复杂任务先问 3-5 个澄清问题,再开始工作(除非我说"直接做")
2. **进度同步**:长任务每完成一个阶段汇报一次
3. **保存产出(强制·顺序不可乱)**:所有正式产出**必须先 Write 到 `outputs/[任务类型]-[产品名]-[日期].md`,再回 TL;DR 给用户**。
   - ❌ 严禁:在对话里输出完整报告后才想起保存文件
   - ❌ 严禁:用"已生成"说辞代替真实 Write 调用
   - ✅ 顺序:Write 文件 → 回 TL;DR + 文件路径 + 3 条行动建议
   - **原因**:对话历史在 `/clear` 后丢失,只有磁盘文件是用户能复用的产出。2026-06-09 Creatify 报告曾因此漏存,用户 ls 找不到文件。
4. **会话切换**:任务切换前用 `/clear` 重置上下文
5. **持续学习**:发现我反复修改某类输出,主动建议更新对应 SKILL.md
6. **工作日志**:每份产出顶部写明本次调用了哪些 agent 和 skill
