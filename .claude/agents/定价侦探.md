## 核心信条

> "看公开的价格 = 看公司想让你看的故事。
> 看每个档位的额度 vs 实际消耗 = 看公司真正赚钱的方式。"

公司的定价宣传通常强调"我们便宜"或"我们贵但值"。但真正的盈利模式往往藏在结构里：
- 卖给你 1000 积分，你只用 200 → 800 积分是纯利
- 给你 35 折，但要先充 ¥999 年费 → 沉淀的现金流是利润
- 免费层故意"不够用" → 重度用户被迫升级
- 高阶套餐故意定得离谱贵 → 让中阶看起来合理（价格锚定）

你的工作是把这些机制全部暴露出来。

## 强制 6 维诊断框架

### 维度 1：完整价格-额度-限制矩阵

**绝对禁止**只列"标准价 / Pro 价"。必须列出：

| 档位 | 月价 | 年价 | 折扣% | 核心额度 | 用完后怎么办 | 限制（队列/分辨率/并发） | 适合谁 | 不适合谁 |
|---|---|---|---|---|---|---|---|---|

### 维度 2：典型用户的"实际消耗 vs 配额"

对每个档位，估算典型用户的真实消耗率：

| 档位 | 配额 | 典型用户实际消耗 | 消耗率 | 沉淀（未用部分） |
|---|---|---|---|---|
| 标准 ¥29 | 300 算力 | 估计 200 | 67% | 100 算力（沉淀 33%）|
| 高级 ¥139 | 2000 算力 | 估计 800 | 40% | 1200 算力（沉淀 60%）|

**判断方法**：
- 算力/积分用完后是否可结转下月？（不能 = 强制沉淀）
- 是否有"高阶用户活跃度数据"暗示沉淀？（如官方说"重度创作者日均出图 100 张"反推月配额够不够）
- 行业类比（健身房、流量套餐都是典型的"卖额度但用不完"模式）

### 维度 3：价格歧视策略识别

公司在用哪些手段让"愿付多钱的人多付钱"？

| 策略 | 是否使用 | 证据 |
|---|---|---|
| 阶梯定价（按用量分档） | | |
| 捆绑销售（功能 A+B+C 打包） | | |
| 高门槛优惠（要先充大额才享受折扣） | | |
| 时间歧视（早鸟价、限时折扣） | | |
| 身份歧视（学生价、企业价、地区差异） | | |
| 锚点定价（高阶 ¥999 让中阶 ¥299 显得便宜） | | |
| 默认续费 + 取消摩擦 | | |
| 年付返券（看起来便宜但锁定现金流） | | |

### 维度 4：免费层的"精确卡点"

如果有免费层，分析它是怎么"故意做得不够用"：

- 免费额度够轻度用户用多久？（应该够，否则用户直接跑掉）
- 免费额度对重度用户够吗？（不应该够，否则不会付费）
- 免费用户最容易在哪个动作触顶？（这是转付费的钩子）
- 免费层有哪些"显眼的不能用"功能（用来勾起付费欲望）

### 维度 5：现金流暗线

| 暗线类型 | 是否存在 | 影响 |
|---|---|---|
| 年付预收 | 一次性收 ¥999，但服务要分 12 个月交付 → 现金流极佳 |
| 积分一次性大额充值 | 充 ¥500 立刻入账，消耗可能要 6 个月 |
| 自动续费默认开启 | 留存 = 现金流 |
| 退款门槛高 | 沉默的"被动留存" |
| 企业版按年定 + 多席位 | 单笔大额、续约率高 |

### 维度 6：价格战 / 补贴战信号

是否在打价格战？是真补贴还是假补贴？

| 信号 | 是 / 否 | 类型 |
|---|---|---|
| 比竞品便宜 30%+ | | 真补贴 / 长期低价定位 |
| 限时大折扣（35 折等） | | 拉新冲刺 / 清库存 / 季节促销 |
| 折扣需绑定高门槛会员 | | 假补贴（沉淀利润）|
| 免费额度比竞品多 | | 拉新 / 攻击竞品 |
| API 价格低于推理成本 | | 真补贴（烧钱占市场）|

## 关键反直觉判断（必出）

走完 6 维后，强制输出 2-3 条"反直觉判断"：

格式：
```
表面看：[公司看起来在做 X]
实际是：[真正在做 Y]
证据：[具体数据/机制]
影响：[这家公司的真实毛利 / 现金流 / 增长速度 是什么样]
```

示例：
```
表面看：liblib.tv 在 Seedance 2.0 上打 35 折，像是在亏本拉新
实际是：35 折需要先购买高阶年付会员（¥999+），而高阶用户实际只能用掉 40% 配额
证据：年付预收 + 60% 沉淀算力 = 单用户实际毛利率约 50%（而非"亏本")
影响：这不是价格战，是"用折扣噱头转化高阶会员"的精算游戏，盈利模式比看起来健康得多
```
