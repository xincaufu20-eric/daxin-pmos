# 商业画布 SKILL

> 基于 9 格商业模式画布经典框架。
> 输出目标：让任何人 5 分钟看懂这个业务怎么运作。
> **协同**:涉及定价细节时,调用「定价反推」补充;涉及创始人战略时,调用「创始人画像」补充。

## 逐格分析法

### 1️⃣ Customer Segments（客户细分）
**问**：我们为谁创造价值？

- 不只列"谁"，要列"谁的什么场景"
- 区分付费用户 vs 使用用户（如广告模式：用户用、广告主付）
- 量化每个细分的规模和价值

**输出格式**：

| 细分群体 | 占比 | 核心需求 | 付费意愿 |
|---|---|---|---|
| | | | |

### 2️⃣ Value Propositions（价值主张）
**问**：我们解决什么痛点？提供什么独特价值？

- 强制用"对比"格式：相比 [现有方案]，我们 [独特优势]
- 区分**功能性价值**（省时省钱）、**情感性价值**（自豪 / 归属感）、**社会性价值**（身份认同）

### 3️⃣ Channels（渠道）
**问**：如何触达客户？

5 阶段渠道：
- **认知**：用户怎么知道我们？
- **评估**：用户怎么评估我们好不好？
- **购买**：怎么完成购买？
- **交付**：产品/服务怎么交付？
- **售后**：怎么持续服务？

### 4️⃣ Customer Relationships（客户关系）
**问**：和客户是什么样的关系？

类型清单：
- 个性化服务（高 ARPU 业务）
- 自助服务（SaaS、电商）
- 自动化服务（推荐算法、机器学习）
- 社区（小红书、Reddit）
- 共创（Wikipedia、开源）

### 5️⃣ Revenue Streams（收入来源）
**问**：钱怎么来？

收入类型：
- 一次性销售（硬件、SaaS 永久授权）
- 订阅费（SaaS、流媒体）
- 使用费（云服务、API 调用）
- 广告费（内容平台）
- 抽佣 / GMV %（电商、O2O）
- 数据 / 排名（搜索、推荐位）

每条必须标：
- 占总收入比例
- 增长率
- 毛利率

### 6️⃣ Key Resources（关键资源）
**问**：我们靠什么资源运作？

类型：
- 物理资源（工厂、店铺、服务器）
- 知识资源（专利、品牌、数据）
- 人力资源（核心团队）
- 财务资源（现金、信用额度）

### 7️⃣ Key Activities（关键活动）
**问**：我们日常在做什么？

类型：
- 生产（制造业）
- 平台运营（互联网）
- 问题解决（咨询、服务）
- 销售（B2B）

### 8️⃣ Key Partners（关键合作）
**问**：靠谁的帮助才能运作？

类型：
- 战略联盟（Apple ↔ TSMC）
- 供应商关系（亚马逊 ↔ 物流商）
- 协同创业（Uber ↔ 司机）

### 9️⃣ Cost Structure（成本结构）
**问**：钱花在哪？

必填项：
- 固定成本（人力、租金、服务器基础）
- 变动成本（按使用量 / 客户数）
- 规模经济性（用户翻倍时成本翻几倍？）

## 增强分析：Unit Economics（单位经济模型）

对 SaaS / 互联网产品，**必做**：

| 指标 | 数值 | 标杆 |
|---|---|---|
| CAC（获客成本） | | < 1/3 LTV 为佳 |
| LTV（客户终身价值） | | 越高越好 |
| LTV / CAC ratio | | > 3 为健康 |
| CAC Payback Period | | < 12 个月为佳 |
| Gross Margin | | SaaS > 70% |
| Churn Rate（月度流失） | | < 5% |
| Magic Number (SaaS) | | > 1 健康，> 1.5 需加速投入 |
| Rule of 40 | 增长率% + 利润率% | > 40 为优秀 |

## 商业模式诊断清单

填完画布后，回答这 10 个诊断问题：

1. **谁是真正的客户？**（付钱的 vs 使用的可能不同）
2. **核心价值能用一句话说清吗？**（说不清就是不清楚）
3. **客户为什么选我们而不是替代品？**（包括"不做"的替代）
4. **我们最薄弱的格子是哪个？**（薄弱处是攻击点也是机会点）
5. **网络效应在哪？**（用户多了对其他用户更有价值吗？）
6. **规模经济性如何？**（用户翻倍，成本翻几倍？）
7. **谁拥有用户关系？**（是我们 vs 是渠道/平台）
8. **数据飞轮在哪？**（数据 → 更好产品 → 更多用户 → 更多数据）
9. **最大的单点失败风险是什么？**（一个合作方撤了会怎样？）
10. **5 年后这个模式还成立吗？**

## 反模式

❌ 9 格都写"创新驱动"、"用户至上"这种空话
❌ 没有具体数字（必须有 ARPU、毛利率、用户量等）
❌ 缺乏对比（必须说清"为什么不选替代品"）
❌ 忽略 Cost 和 Revenue 的实际数据
