# 产品交互走查 SKILL

> 综合方法论:
> - Jakob Nielsen 10 Usability Heuristics(可用性十大启发式)
> - Growth.Design 的产品心理学走查(行为经济学/认知偏差视角)
> - Baymard Institute 的流程审计(电商/SaaS 流程逐步审视)
> - HEART 框架(Google 的用户体验度量)
> 输出目标:让 PM 看完就知道"为什么这个产品体验好/差",每个判断都有具体页面证据。

---

## Phase 0:明确走查目的

```
开始走查前,先明确:

1. 走查目的?
   A. 学习借鉴(他们做对了什么我能抄)
   B. 找漏洞(我要攻击竞品哪里)
   C. 自我审视(我们产品的体验问题)
   D. 写体验报告(给团队/老板看)

2. 走查深度?
   A. 快速走查(只看核心 3-5 个页面/流程,20 分钟)
   B. 标准走查(完整核心流程 + 启发式打分,1 小时)
   C. 深度审计(全流程 + 心理学拆解 + 改进建议,2-3 小时,推荐用于正式报告)

3. 关注角色?
   A. 首次访问者(对产品一无所知)
   B. 付费决策者(在考虑要不要付钱)
   C. 重度使用者(已经用了一段时间)
   D. 全部 3 个角色都走一遍
```

## Phase 1:抓取产品当下状态(强制)

**绝对不能跳过**:必须用 WebFetch 抓取或调用「产品体验员」子智能体走查。

抓取范围:
- 未登录首页
- 定价页
- 注册/登录页
- 当前活动/促销页
- 帮助中心首页
- Blog/Changelog(看最近 3 个月更新)

如果产品无法访问:**告知用户并请求提供截图,不允许用搜索结果替代**。

---

## Phase 2:三大框架并行走查

### 框架 A:Nielsen 10 启发式打分

对产品逐项打分(1-5 分,1=严重违反,5=完美遵守):

| # | 启发式 | 含义 | 打分 | 证据(具体页面/操作) | 改进建议 |
|---|---|---|---|---|---|
| 1 | 系统状态可见性 | 用户能否随时知道系统在做什么(加载/进度/反馈) | | | |
| 2 | 系统与真实世界匹配 | 用语/隐喻/概念是否符合用户已有认知 | | | |
| 3 | 用户控制与自由 | 是否提供"撤销"、"返回"、"退出"等控制 | | | |
| 4 | 一致性与标准 | 同一概念在不同页面表现一致;遵守平台惯例 | | | |
| 5 | 错误预防 | 是否在用户犯错前预防(而非犯错后补救) | | | |
| 6 | 识别优于回忆 | 选项/信息可见,无需用户记忆 | | | |
| 7 | 灵活性与效率 | 新手能用,老手有快捷方式 | | | |
| 8 | 美学与极简设计 | 界面无冗余信息 | | | |
| 9 | 帮助用户识别/恢复错误 | 错误信息明确,提供解决方案 | | | |
| 10 | 帮助与文档 | 必要时易于找到帮助 | | | |

**输出强制要求**:
- 每项打分必须有"具体页面 + 具体操作"证据
- 不允许"还行"、"一般"这种模糊评价
- Top 3 最高分和 Top 3 最低分必须标出

### 框架 B:Growth.Design 心理学走查

识别产品在哪些环节用了行为经济学/心理学技巧:

| 心理学机制 | 是否使用 | 在哪个页面/动作 | 效果评估 |
|---|---|---|---|
| **社会证明** (Social Proof) | | "X 万用户使用"、用户头像墙、好评展示 | |
| **稀缺性** (Scarcity) | | "仅剩 3 个名额"、库存告急 | |
| **紧迫感** (Urgency) | | 倒计时、限时折扣 | |
| **锚定效应** (Anchoring) | | 划掉的"原价"、高阶套餐做锚点 | |
| **损失厌恶** (Loss Aversion) | | "错过本次永远没有"、"会员到期前续费" | |
| **承诺一致性** (Commitment) | | 让用户先做小承诺(免费试用)再推大承诺 | |
| **互惠** (Reciprocity) | | 免费送资料、免费试用、礼包 | |
| **默认效应** (Default Bias) | | 默认勾选"自动续费"、"接收邮件" | |
| **峰终定律** (Peak-End Rule) | | 关键节点(注册成功、付款成功)的庆祝/反馈 | |
| **进度效应** (Progress Effect) | | 注册进度条、任务完成度 | |
| **禀赋效应** (Endowment) | | "你的"个性化展示,让用户感到拥有 | |
| **沉没成本** (Sunk Cost) | | 让用户已经投入(填表/上传)后才出现门槛 | |

**输出**:
- 用了几种机制?
- 使用最巧妙的 1-3 个(可以学习的)
- 使用过激/有道德风险的 1-3 个(暗黑模式 Dark Pattern)

### 框架 C:Baymard 流程审计(关键流程逐步打分)

选取 1-3 个核心流程,做"步骤级"审计:

典型流程:
- 注册流程
- 首次使用流程(Onboarding)
- 付款流程
- 取消订阅流程(看产品是否"易进难出")
- 客户服务接入流程

对每个流程:

| 步骤 # | 用户动作 | 系统反馈 | 阻力点 | 心理状态 | 改进机会 |
|---|---|---|---|---|---|
| 1 | | | 🟢无/🟡轻/🔴重 | 兴奋/犹豫/烦躁/放弃 | |
| 2 | | | | | |

**强制问题**:
- 这个流程共几步?能不能减?
- 最容易放弃的是第几步?为什么?
- 有没有"取消摩擦"(让用户不容易取消订阅)的暗黑设计?

## Phase 3:三角色走查(对应 Phase 0 的角色选择)

对每个目标角色,跑一遍核心流程,记录每个动作的心理活动:

### 角色 1:首次访问者
- 进入首页 3 秒内的反应:[兴奋/疑惑/无感/想关闭]
- 最大疑惑是什么?
- 决定要不要继续探索的关键瞬间是哪一刻?

### 角色 2:付费决策者
- 从产生付费想法到完成付费,共几步?
- 哪一步开始犹豫?
- 价格是否一目了然?有没有"问销售"型隐藏价格?
- 退款政策清楚吗?

### 角色 3:重度使用者
- 高频功能是否有快捷方式?
- 是否有"老用户专属"功能/特权?
- 数据/作品的导出迁移有多容易?(决定离开成本)

## Phase 4:整体评级 + so what

### HEART 五维总评

| 维度 | 评分 (1-5) | 证据 |
|---|---|---|
| **H** Happiness 满意度 | | |
| **E** Engagement 参与深度 | | |
| **A** Adoption 新用户采纳 | | |
| **R** Retention 留存 | | |
| **T** Task Success 任务完成 | | |

### 三类输出(强制)

#### ✅ 可借鉴的 3 件事
1. [具体设计 + 在哪个页面 + 我能怎么抄]

#### ❌ 暗黑模式/糟糕设计 2 个
1. [具体问题 + 用户损失]

#### 🎯 如果我是这家产品的 PM,本周必须改的 3 件事
1. [按 ROI 排序,从易到难]

## 反模式

❌ 没访问产品就开始评价
❌ 只打分不给证据
❌ 用"体验不好"、"挺好的"这种模糊话
❌ 漏掉"暗黑模式"检测(很多看似优秀的产品都有)
❌ 没有"如果是我的产品本周必须改什么"的落地建议

## 协同

- 走查发现增长机制 → `产品考古` 深挖战略
- 走查发现定价异常 → `定价反推`
- 走查多个竞品 → `竞品分析`
- 走查为了写自家产品 PRD → `PRD编写`
