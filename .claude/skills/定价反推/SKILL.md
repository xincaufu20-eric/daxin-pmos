# 定价反推 SKILL

> 哲学:看官方定价 = 看公司想让你看的故事。看每个档位的"配额 vs 实际消耗" = 看公司真正赚钱的方式。
> 输出目标:让 PM/高管能回答"这家公司表面在打价格战实际在赚什么"、"我们该不该跟"。

---

## Phase 0:明确分析对象

```
要分析谁的定价?

1. 目标产品 URL:[必填]
2. 主要竞品 URL(对比定价用):[可选,2-3 个]
3. 分析深度?
   A. 快速诊断(15 分钟,只看明面定价)
   B. 标准反推(45 分钟,推算沉淀利润和真实毛利)
   C. 深度审计(2 小时,含 unit economics 全套推算)
4. 你的目的?
   A. 学习借鉴(我们要不要也这么定)
   B. 攻击竞品(他们的定价漏洞在哪)
   C. 投资判断(这家公司商业模式真健康吗)
   D. 内部汇报
```

## Phase 1:抓取完整定价矩阵(强制实地)

调用「产品体验员」+「定价侦探」实地抓取:

- 公开定价页(所有档位)
- 当前活动页(限时折扣)
- 企业版/团队版定价(可能隐藏在"联系销售")
- 年付页面(年付折扣)
- API 价格页
- 帮助中心里关于"积分"、"配额"、"续费"的 FAQ

## Phase 2:6 维诊断(核心)

### 维度 1:完整价格-额度-限制矩阵

| 档位 | 月价 | 年价 | 年付折扣% | 核心配额 | 配额单位说明 | 用完后 | 队列/分辨率/并发限制 | 适合谁 | 不适合谁 |
|---|---|---|---|---|---|---|---|---|---|

### 维度 2:典型用户消耗 vs 配额(沉淀利润测算)

对每个档位估算:

| 档位 | 配额 | 典型用户实际消耗(估算) | 消耗率 | 沉淀% | 推理依据 |
|---|---|---|---|---|---|

**推理依据可参考**:
- 行业类比(健身房 / 通信套餐都是典型"配额沉淀模式")
- 官方文档中提到的"重度用户日均产出"
- 用户论坛吐槽"配额用不完"或"配额不够"的比例

### 维度 3:价格歧视策略识别

| 策略 | 是否使用 | 证据 | 效果评估 |
|---|---|---|---|
| 阶梯定价(按用量分档) | | | |
| 捆绑销售(功能 A+B+C 打包) | | | |
| 高门槛优惠(要先充大额才享受折扣) | | | |
| 时间歧视(早鸟价、限时折扣) | | | |
| 身份歧视(学生价、企业价、地区差异) | | | |
| 锚点定价(高阶离谱贵让中阶显得合理) | | | |
| 默认续费 + 取消摩擦 | | | |
| 年付返券(看似便宜但锁定现金流) | | | |
| 加价的"硬刚需"功能(如更高分辨率、商用授权) | | | |

### 维度 4:免费层"精确卡点"分析

如果有免费层:
- 免费额度够什么级别的用户用?
- 免费用户最容易在哪个动作触顶?(这是转付费的钩子)
- 免费层故意"显眼地不能用"的功能有哪些?
- 免费 vs 入门付费档,跨越成本(¥/月)是多少?

### 维度 5:现金流暗线

| 暗线 | 是否存在 | 影响 |
|---|---|---|
| 年付预收(一次性大额) | | 现金流极佳,实际服务要分 12 个月交付 |
| 积分一次性大额充值 | | 充值即入账,消耗可能 6 个月 |
| 默认自动续费 | | 留存 = 现金流稳定 |
| 高退款门槛 | | 被动留存 |
| 企业版多席位锁定 | | 单笔大额、续约率高 |

### 维度 6:价格战 / 补贴战信号

| 信号 | 是 / 否 | 真假补贴判断 |
|---|---|---|
| 比竞品便宜 30%+ | | 真补贴/长期低价定位 |
| 限时大折扣(35 折等) | | 拉新冲刺/清库存/季节促销 |
| 折扣需绑定高门槛会员 | | 假补贴(沉淀利润) |
| 免费额度比竞品多 | | 拉新/攻击竞品 |
| API 价格低于推理成本 | | 真补贴(烧钱占市场) |
| 新用户专属价(老用户不享受) | | 拉新优先,留存让位 |

## Phase 4:反直觉判断(最值钱)

```
表面看:[公司看起来在做 X]
实际是:[真正在做 Y]
证据:[具体数据 + 推理]
影响:[对市场/对竞品/对我们的真实意义]
```
