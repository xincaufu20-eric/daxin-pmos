# 竞品分析 SKILL

> 输出目标：可执行的差异化战略，不是 feature list 复述。
> **强制**:每个被对比的产品都必须由「产品体验员」实地访问,关键定价点由「定价侦探」反推,用户评价由「用户声音侦察员」挖掘。

---

## 触发后的标准动作

### Step 1：澄清竞品范围（必做）

```
开始竞品分析前，先确认 3 件事：

1. 你的产品是什么？解决的核心问题是什么？
   （如果 CLAUDE.md 已说明，跳过此问题）

2. 想分析哪些竞品？
   A. 我列了名单：[让用户列出]
   B. 帮我识别（我会推荐 3-5 个直接 + 间接 + 替代品）

3. 分析目的？
   A. 找差异化机会（攻击型）
   B. 防御自己的护城河（防守型）
   C. 写竞品报告给老板/团队
   D. 销售对话辅助（怎么 win deals）
```

### Step 2：竞品识别（如用户选 2B）

按 4 个维度识别竞品（参考 Porter 五力 + JTBD 思路）：

| 类型 | 定义 | 示例（假设我们是 Notion） |
|---|---|---|
| **直接竞品** | 同样产品、同样用户 | Coda, Roam Research |
| **间接竞品** | 不同产品、同样问题 | Evernote, Obsidian |
| **替代品** | 完全不同方案、解决同 JTBD | Google Docs, Word + Excel 组合 |
| **新兴威胁** | 还小但增长快、可能颠覆 | AI Notepad, Mem.ai |

## 分析框架：Aakash 的 Smart/Weak/Implications 三段式

对每个竞品，强制输出 3 段：

### Smart（他们做对了什么）
- 不要写"功能 X"，要写"他们用功能 X 达成了 Y 商业结果"
- 必须有证据（数据 / 用户反馈 / 财报披露）

### Weak（他们的弱点在哪）
- 重点看 G2 / Capterra / 知乎 / 小红书 用户抱怨
- 看 Glassdoor 内部员工反馈
- 关注：onboarding 复杂度、价格不透明、客服差、功能臃肿

### Implications（对我意味着什么）
- 这是最值钱的部分
- 必须答："所以我应该……"
- 三种行动类型：**攻击**（打他弱点）/ **学习**（抄他强点）/ **绕开**（找新空间）

## 输出格式 2：Insight-Impact-Opportunity 表格（Aakash 经典框架）

为每个竞品的关键发现，必须填这个表：

| Insight（洞察） | Impact（影响） | Opportunity（机会） |
|---|---|---|
| 竞品 A 的 onboarding 比我们少 3 步且无需信用卡 | 这可能贡献了他们 20% 更高的 Day 1 留存 | 我们应 A/B 测试去掉信用卡要求，假设：提升 trial signup 15% |
| 竞品 B 的 API 文档差，开发者抱怨多（Stack Overflow 23 个未回答帖子） | 阻碍他们企业市场扩张 | 我们的 API 文档建立护城河，本季度立项 |

---

## 输出格式 3：定位地图（Position Map）

输出一个 2x2 矩阵，把所有竞品（含我们）放上去：

```
                    高 ↑ {选轴 1，如：功能丰富度}
                       |
           竞品B       |       我们
                       |
        ━━━━━━━━━━━━━━━╋━━━━━━━━━━━━━━━ {选轴 2，如：使用门槛}
低 ←  门槛低           |          门槛高  → 高
                       |
           竞品C       |       竞品A
                       |
                    低 ↓
```

**关键发现**：地图上的"空白象限"通常是机会。

---

## 输出格式 4：差异化战略建议

最终输出必须包含 3 类战略选项（参考 Aakash 的 Offensive/Defensive/Innovative 框架）：

### 🗡️ 进攻型机会（Offensive Plays）
攻击竞品已知弱点。
- 例："Coda 的协作权限管理复杂，我们做'一键最简权限'功能"

### 🛡️ 防御型护城河（Defensive Moats）
保护已有优势防止被复制。
- 例："我们的 Notion AI 已有 6 个月数据沉淀，加速 fine-tuning 拉大差距"

### 🚀 创新型蓝海（Innovative Plays）
攻击所有人都没覆盖的需求。
- 例："所有竞品都没做'AI 主动提醒待办从邮件中提取'，市场空白"

## 多视角审查（最后一步）

每份竞品分析完成后，主动建议：

```
建议从竞品视角再 review 一次：

「如果我是竞品 A 的 PM，我看到这份分析会怎么反击？」

这一步常常能暴露我们 strategy 的脆弱性。
```

---

## 反模式（禁止）

❌ **Feature checklist 病**
   - 错误：列一个 20 行的 feature 对勾表
   - 正确：选 5-7 个真正影响购买决策的维度

❌ **二元简化**
   - 错误："我们更好" / "他们更差"
   - 正确："在 X 场景我们强，在 Y 场景他们强，目标用户 Z 更需要 X"

❌ **无 so what**
   - 错误：罗列竞品做了啥
   - 正确：每个发现配一条"我应该……"

❌ **过期信息**
   - 强制：每个数据点标注日期，超过 6 个月的标记 ⚠️ 待更新

❌ **遗漏间接竞品**
   - 用户真正的选择往往是"用 Excel 凑合" "买台 iPad" 这类替代品
   - 必须问"如果他们不用我们的产品，他们用什么？"

## 经典 AI 加速 prompt（嵌入 skill 中）

调研竞品 G2 / Capterra / 知乎评论时，使用此 prompt：

```
扮演一个资深 PM。分析以下 [产品名] 的 20 条用户评论。
总结：
1. Top 5 最常见的用户抱怨
2. Top 3 最被称赞的功能
3. 每个抱怨对应一个潜在的产品机会
用 markdown 表格输出。
```
